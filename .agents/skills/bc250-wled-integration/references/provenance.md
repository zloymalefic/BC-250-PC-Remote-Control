# Code provenance and licensing

WLED is distributed under EUPL v1.2. The BC-250 repository may have different or unspecified licensing terms.

- Record the exact source repository, commit, file, and algorithm when adapting WLED code.
- Prefer reimplementation from documented behavior and interfaces over copying substantial source blocks.
- Preserve required copyright and license notices for copied code.
- Do not assume code can be relicensed merely because it was modified.
- Mark substantial AI-generated WLED code with the comments required by WLED's `AGENTS.md`.
- Treat licensing conclusions as a release/legal review item when distributing combined or derivative firmware.

This reference is an engineering provenance checklist, not legal advice.
